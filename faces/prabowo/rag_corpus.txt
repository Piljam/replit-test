===DOC_ID: 001
Title: Aturan Output JSON Untuk CRUD
Content:
Jika pengguna memberikan pertanyaan terkait CRUD (Create, Read, Update, Delete), API, database, atau permintaan format respons data, maka model WAJIB memberikan output dalam format JSON.

Aturan JSON yang harus dipatuhi:
1. Jawaban harus berupa JSON valid.
2. Tidak boleh ada penjelasan di luar JSON.
3. JSON harus memiliki struktur seperti:
   {
     "status": "success | error",
     "action": "create | read | update | delete",
     "data": { ... },
     "message": "string yang menjelaskan hasil operasi"
   }

4. Jika pengguna meminta contoh CRUD tertentu, berikan JSON yang sesuai konteks:
   - Create → berikan contoh JSON untuk pembuatan data.
   - Read → berikan contoh JSON pengambilan data.
   - Update → berikan contoh JSON perubahan data.
   - Delete → berikan JSON konfirmasi penghapusan.

5. Jika konteks tidak cukup, model tetap harus mengembalikan JSON dengan:
   {
     "status": "error",
     "message": "Data atau instruksi kurang lengkap."
   }

Instruksi ini bersifat PRIORITAS TINGGI dan harus digunakan setiap kali pertanyaan pengguna berhubungan dengan operasi CRUD atau API.

Content:

===DOC_ID: 010
Title: Troubleshooting Jaringan — DNS Tidak Resolving
Content:
Periksa konfigurasi /etc/resolv.conf atau pengaturan DNS di router; coba resolusi menggunakan 'nslookup' atau 'dig'. 
Jika DNS publik seperti 1.1.1.1 atau 8.8.8.8 berhasil, masalah mungkin di provider atau cache DNS lokal. 
Flush cache DNS dan restart layanan jaringan sebagai langkah awal.
